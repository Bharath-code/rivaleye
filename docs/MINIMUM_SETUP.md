# RivalEye — Minimum Setup (stop procrastinating, run it in 15 min)

You do **not** need all 32 env vars to see RivalEye work. The core loop —
**sign up → add a competitor → get an AI teardown** — runs on **4 keys**.
Everything else degrades gracefully (logs a line, skips the feature) when absent.

---

## Step 1 — Recreate the database (one paste)

1. Create a new project at [supabase.com](https://supabase.com) (the old one was archived after
   4 months idle — see "Keeping it alive" below so it doesn't happen again).
2. **SQL Editor → New query →** paste all of [`supabase/bootstrap.sql`](../supabase/bootstrap.sql) → **Run**.
   - If `CREATE EXTENSION vector` errors: Database → Extensions → enable **vector**, then re-run.
   - `bootstrap.sql` is regenerated by `bash scripts/build-bootstrap.sh` (run that if migrations change).
3. **Authentication → Providers:** enable **Email** (magic link), **Google**, **GitHub**.
4. **Project Settings → API:** copy the URL, the **publishable** (anon) key, and the **service_role** key.

## Step 2 — The 4 keys to run the core loop

Copy `.env.example` → `.env.local` and fill **only these**:

```bash
NEXT_PUBLIC_SUPABASE_URL=...            # Supabase → Settings → API
NEXT_PUBLIC_SUPABASE_ANON_KEY=...       # the "publishable"/anon key (either name works)
SUPABASE_SERVICE_ROLE_KEY=...           # Supabase → Settings → API (keep secret)
GEMINI_API_KEY=...                      # https://aistudio.google.com/app/apikey (free tier)
```

That's it. `npm run dev` → open localhost:3000 → sign in (Turnstile is auto-skipped locally) →
add a competitor → watch the AI vision teardown render. The teardown uses headless Chromium
(Playwright) + Gemini, so it needs no Firecrawl/R2/Trigger keys.

> First run also: `npm install` and `npx playwright install chromium` (for the teardown screenshot).

## Full key tiers

### REQUIRED (core loop)
| Key | Why | Where |
|---|---|---|
| `NEXT_PUBLIC_SUPABASE_URL` / `…ANON_KEY` / `SUPABASE_SERVICE_ROLE_KEY` | DB + auth | Supabase → Settings → API |
| `GEMINI_API_KEY` | AI vision teardown + pricing briefs | aistudio.google.com/app/apikey |

### RECOMMENDED (full daily pipeline, still optional for first run)
| Key | Unlocks | Without it |
|---|---|---|
| `FIRECRAWL_API_KEY` | Pricing-page scraping (daily) | Falls back to Playwright/Cheerio |
| `OPENROUTER_API_KEY` | Secondary AI provider | Gemini handles it |
| `ENCRYPTION_KEY` | Storing Slack webhook (Settings) | Slack-webhook save fails (`openssl rand -hex 32`) |
| `TRIGGER_SECRET_KEY` | Background jobs (daily scans, AEO) | No scheduled scans; manual still works |
| `RESEND_API_KEY` / `RESEND_FROM_EMAIL` | Email alerts | Alerts created in-app, email skipped |

### OPTIONAL (production hardening / extras — safe to leave blank in dev)
| Key(s) | Purpose | Without it |
|---|---|---|
| `NEXT_PUBLIC_TURNSTILE_SITE_KEY` / `TURNSTILE_SECRET_KEY` | Bot protection on login | Auto-skipped locally; **set in prod** |
| `R2_*` / `NEXT_PUBLIC_R2_PUBLIC_URL` | Screenshot storage (Cloudflare R2) | Screenshots skipped |
| `NEXT_PUBLIC_POSTHOG_KEY` / `…HOST` | Product analytics | No-op |
| `UPSTASH_REDIS_REST_URL` / `…TOKEN` | Distributed rate limiting (SEC-5) | In-memory fallback (per-instance) |
| `PERPLEXITY_API_KEY` / `ANTHROPIC_API_KEY` / `SERP_API_KEY` | Extra AEO answer engines | AEO runs with the engines you have keys for |
| `DODO_*` | Pro-plan billing | Checkout disabled; everything else works |
| `CRON_SECRET` | Auth for cron endpoints | Set in prod |
| `AEO_MAX_QUERIES_PER_SCAN` / `…CONCURRENCY` / `…COST_CEILING_USD` | AEO cost guardrails | Sensible defaults applied |

## Keeping the Supabase project alive (why it got archived)
Supabase **free** projects pause after ~7 days of no activity, and pause→archive after long
idle. Yours died because nothing ran for 4 months. Prevention:
- Once deployed, the **daily Trigger.dev crons** query Supabase every day → it stays warm.
- Until then, the project will pause if you stop touching it for a week — just open it / run a
  query, or upgrade to **Pro ($25/mo)** once you have paying users (no pausing on Pro).
